# Preguntas

- ¿Cuáles son las principales diferencias entre Haskell y RACKET?

A pesar de que ambos son lenguajes funcionales, tienen enfoques distintos. Racket es mas dinámico y flexible que permite efectos secundarios y programación procedimental, Haskell es un lenguaje en donde los efectos secundarios se gestionan estrictamente.
En general Racket es excelente para crear nuevos lenguajes mientras que Haskell se enfoca en la creación de software robusto.

- ¿Cuáles son las principales diferencias entre Haskell y Java? 

Java es un programa con un paradigma de programación imperativo con una filosofía de orientación a objetos mientras que Haskell es declarativo-funcional puro fuertemente tipado, esto quiere decir que detecta errores de tipos en tiempo de compilación.

- ¿Por qué Haskell no ha alcanzado una adopción significativa en la industria del software?

En general es por la curva de aprendizaje que existe en el lenguaje ya que llega a ser dificil para principiantes. La sintaxis que maneja es muy diferente a la mayoria de lenguajes de programación que suelen ser mas usados.
Haskell utiliza por defecto la evaluación perezosa, y si bien esto suele ser una gran ventaja, a veces puede ser perjudicial. Usar la evaluación perezosa de forma ingenua en situaciones importantes puede generar cuellos de botella (punto de congestión en un proceso, sistema o línea de producción que limita la capacidad y el rendimiento general debido a una capacidad inferior o sobrecarga en una etapa específica) innecesarios en el rendimiento.

- Si pudieras realizar una simple analogía entre Git y Github ¿Cuál se te vendría a la mente?

Podriamos decir que Git es un nuevo platillo en el que trabaja un chef para el menú de un restaurante que al ser finalizado este se supervisara y finalmente se agregara al menú del restaurante junto con los demas platillos que vendria siendo github.
